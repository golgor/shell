## 8. Capslock detection not working
**Effort**: Smallâ€“Medium (diagnosis done)

Capslock/numlock state is always reported as off. The shell reads lock state from the keyboard marked `main` in Hyprland's device list (`Hypr.keyboard?.capsLock`). The problem: `main: true` is set on `hl-virtual-keyboard-fcitx5` (an input method virtual keyboard), not the physical keyboard (`logitech-ergo-k860`). The virtual keyboard always reports `capsLock: false`.

The detection mechanism works via:
1. Hyprland binds `Caps_Lock`/`Num_Lock` keys to a global shortcut (`caelestia:refreshDevices`)
2. On key press, `HyprExtras.refreshDevices()` re-queries `hyprctl devices -j`
3. `HyprKeyboard.capsLock` is read from the JSON for the keyboard where `main: true`

**Options**:
1. **Fix keyboard selection** â€” Instead of `keyboards.find(kb => kb.main)`, find the physical keyboard by filtering out virtual/hotkey keyboards. Could match by name pattern or check multiple keyboards.
2. **Aggregate lock state** â€” OR the capsLock state across all keyboards: `keyboards.some(kb => kb.capsLock)`
3. **Hyprland config** â€” Investigate if Hyprland can be configured to set `main` on the correct device.

**Key files**:
- `services/Hypr.qml:27` â€” `keyboards.find(kb => kb.main)` â€” keyboard selection
- `plugin/src/Caelestia/Internal/hyprdevices.hpp` â€” `HyprKeyboard` class, reads from `hyprctl devices`
- `plugin/src/Caelestia/Internal/hyprdevices.cpp` â€” IPC JSON parsing
